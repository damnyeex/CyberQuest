# CyberQuest – Платформа обучения кибербезопасности через CTF-задачи

**CyberQuest** — это полнофункциональная веб-платформа для практического обучения специалистов по информационной безопасности.  
Основной упор сделан на **CTF-формат (Capture the Flag)**, геймификацию и адаптивную систему прогресса.

---

## 🎯 Ключевые возможности
- **Каталог курсов и CTF-задач** с категоризацией по сложности и тегам
- **Геймификация и прогресс**: очки опыта, бейджи, рейтинги и ежедневные цели
- **Ролевая модель**: студент, учитель, администратор
- **Интерактивный CTF-тренажёр** с подсказками и проверкой флагов
- **Административная панель** для управления пользователями, задачами и логами
- **Защищённые маршруты** с проверкой JWT‑токенов и ролей

---

## 🧱 Архитектура платформы

### Общая схема взаимодействия
- **Клиентская часть** – одностраничное приложение (SPA) на **Next.js 14** (App Router)
- **Серверная часть** – REST API на **Flask** с использованием **Flask-SQLAlchemy** и **PyJWT**
- **База данных** – **PostgreSQL**, ORM-модели описаны с помощью SQLAlchemy
- **Аутентификация** – JWT-токены (access + refresh), хеширование паролей

### Фронтенд

| Аспект | Решение |
|--------|---------|
| **Фреймворк** | Next.js 14 (App Router), React 18 |
| **Стилизация** | SCSS-модули с CSS custom properties |
| **Архитектура** | Feature-Sliced Design (FSD) – разделение на `app`, `pages`, `widgets`, `features`, `shared` |
| **Управление состоянием** | React Context + провайдеры (авторизация, уведомления) |
| **Маршрутизация** | Файловая система Next.js, динамические маршруты (`/courses/[id]`) |
| **Защита роутов** | Компонент `ProtectedRoute` с проверкой авторизации и ролей |
| **API-слой** | Кастомные сервисы на основе `axios`, разделённые по сущностям (auth, challenges, users, admin…) |
| **Анимации** | IntersectionObserver через кастомный хук `useScrollAnimation` |

### Бекенд

| Аспект | Решение |
|--------|---------|
| **Фреймворк** | Flask |
| **ORM** | Flask-SQLAlchemy |
| **Аутентификация** | JWT (access-токены), декораторы `@token_required`, `@admin_required` |
| **Хеширование флагов** | `generate_password_hash` / `check_password_hash` |
| **Кросс-доменные запросы** | `flask-cors` с настройкой credentials |
| **Безопасность** | CORS-политика, хеширование паролей, ограничение попыток входа (на уровне модели) |
| **База данных** | PostgreSQL, миграции через `db.create_all()` |

## 🧠 Ключевые архитектурные решения

1. **Изоляция бизнес-логики**  
   Серверная часть содержит слой API (`routes/api.py`), модели (`models.py`) и утилиты аутентификации (`auth.py`). Клиентские API‑сервисы зеркально повторяют эндпоинты сервера.

2. **Гибкая система прогресса**  
   Для каждого пользователя ведётся три уровня статистики: общий прогресс, прогресс по категориям и история решений. Это позволяет строить персонализированные рекомендации.

3. **Безопасность CTF-задач**  
   Флаги хранятся в виде bcrypt-хешей, что исключает их утечку даже при доступе к базе. Подсказки имеют динамическую стоимость (XP) и могут быть запрошены через API.

4. **Расширяемая ролевая модель**  
   Система ролей (`student`, `teacher`, `admin`) реализована через связь «многие ко многим», что упрощает добавление новых ролей и прав.

5. **Масштабируемая структура фронтенда**  
   FSD-архитектура позволяет легко добавлять новые страницы, виджеты и фичи без изменения существующего кода. Все стили изолированы через CSS-модули.

6. **Универсальный UI-кит**  
   Переиспользуемые компоненты (Button, Modal, ProgressBar) вынесены в `shared/UI`, что ускоряет разработку и поддерживает единый дизайн.

## 🔒 Безопасность

- Все пароли хешируются с помощью `werkzeug.security.generate_password_hash`
- Токены доступа имеют ограниченное время жизни (24 часа)
- Защита от подбора: счётчик неудачных попыток входа (поле `failed_login_attempts`)
- Кросс-доменные запросы строго ограничены через CORS (только доверенные origin)
- Флаги задач хранятся в виде хешей, проверка через `check_password_hash`
